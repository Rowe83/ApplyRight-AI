# v0.1.0 — 开源自托管版

## 这是什么

ApplyRight AI 是一款**可自托管**的简历与岗位 JD 智能匹配工具：在本地运行 Next.js，使用你自己的 AI API，所有数据落在 `data/` 目录。

## 功能亮点

- 上传 PDF 简历，粘贴 JD，一键 AI 匹配分析
- 匹配度、缺口关键词、润色建议与章节级 Diff 对比
- 匹配历史回放
- 我的简历管理
- 多 AI 厂商：**OpenAI**、**DeepSeek**、**Anthropic**、**Ollama**（BYOK，Key 仅配置在 `.env`）

## 快速开始

```bash
git clone https://github.com/Rowe83/ApplyRight-AI.git
cd ApplyRight-AI
cp .env.example .env.local
# 编辑 .env.local，填入所选厂商的 API Key
npm install
npm run dev
```

打开 http://localhost:3000 即可使用（无需登录）。

## 配置说明

| 变量 | 说明 |
|------|------|
| `DATA_DIR` | 数据目录，默认 `./data` |
| `AI_PROVIDER` | `openai` / `deepseek` / `anthropic` / `ollama` |
| `AI_MODEL` | 模型名称，如 `deepseek-chat` |
| `DEEPSEEK_API_KEY` / `DEEPSEEK_BASE_URL` | DeepSeek |
| `OPENAI_API_KEY` | OpenAI（可选 `OPENAI_BASE_URL`） |
| `ANTHROPIC_API_KEY` | Anthropic |
| `OLLAMA_BASE_URL` | 本地 Ollama，默认 `http://127.0.0.1:11434` |

应用内 **AI 设置**（`/dashboard/settings`）可切换厂商与模型；配置写入 `data/config.json`，**API Key 不会**写入该文件。

## 数据与备份

- 简历 PDF：`data/resumes/{id}/`
- 匹配历史：`data/history/`
- 备份方式：复制整个 `data/` 目录

## 安全提示

本版本面向**本机或可信内网**使用，不包含用户登录与访问控制。请勿将未加防护的实例直接暴露到公网。
